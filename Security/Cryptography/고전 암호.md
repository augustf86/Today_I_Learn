# 고전 암호

* **고전 암호**(classical cipher)
    - 컴퓨터와 같은 고성능 연산 장치가 발명되기 전에, 비교적 간단한 기계와 손 등으로 암﹒복호화를 수행하던 암호 <br/> &nbsp;&nbsp; = 컴퓨터가 등장하기 전에 쓰인, 비트가 아닌 문자를 다루는 암호
        + 현대적인 암호들보다 훨씬 간단하고 대부분 컴퓨터를 사용하면 쉽게 복호화되기 때문에 현대에는 사용되지 않음
    - 고전 암호는 일반적으로 치환과 전치의 방법으로 설계됨
        | 용어 | 설명 |
        |---|------|
        | 치환(Substitution) | 평문의 문자를 다른 문자로 바꾸는 것 |
        | 전치(Transposition) | 평문 문자들의 위치를 바꾸는 것 |

## 치환 암호(Substitution Cipher)
평문의 각 문자를 약속된 다른 문자로 치환하는 암호

<br/>

### 단일 문자 치환 암호(Monoalphabetic Substitution Cipher)
* 복호화를 위한 치환의 대응 관계가 **일대일 대응**임

<br/>

* 단일 문자 치환 암호의 예시
    - 카이사르(Caesar Cipher) = 시저 암호
        | | 설명 |
        |---|------|
        | 암호화 | 평문의 각 알파벳을 일정한 거리만큼 밀어서 다른 알파벳으로 치환함 |
        | 복호화 | 암호문의 각 문자를 다시 원래 위치로 밀어서(암호화와 반대 방향으로 같은 거리만큼 밀어서) 평문을 구함 |
        | 키(Key) | 알파벳을 밀어낸 횟수 <br/> &nbsp;&nbsp; → 송신자와 수신자는 '몇 칸을 밀 것인지' 사전에 합의해야 통신이 이뤄질 수 있음 |
        | 키 공간 | 가능한 모든 키의 집합(Key Space) = 26 → 알파벳이 총 26자이기 때문 |
        | 해독 | 알파벳을 밀어낸 횟수(= 키)만 알면 해독할 수 있음 <br/> &nbsp;&nbsp; - 가능한 자리 이동 값의 가짓수가 25이므로 25번 시도하면 풀 수 있음 |

        <br/>
        <img width="2560" alt="카이사르 암호" src="https://github.com/augustf86/Today_I_Learn/assets/122844932/68c2eabe-ef2b-42e4-95f6-0f5e10fbaa6e"><br/><br/>

    - [셜록 홈즈]에 나오는 춤추는 인형 암호
        | | 설명 |
        |---|------|
        | 암호화 | 사람 한 명이 글자 하나에 대응됨 → 모든 알파벳을 서로 다른 기호와 무작위로 일대을 대응시켜 치환함 |
        | 키 공간 | 26! (무작위로 일대을 대응시켜 치환) |
        | 해독 | 26!의 키 공간은 현대의 컴퓨터로도 탐색하기 어렵기 때문에 안전한 것처럼 보임 <br/> &nbsp;&nbsp; → **통계적 특성이 유지된다는 특성** 때문에 충분히 큰 키 공간에도 불구하고 쉽게 해독될 수 있음 |

        <br/>

    - 난수표나 코드북을 이용한 단일 치환 암호 (현재도 종종 사용되고 있음)
        | | 설명 |
        |---|------|
        | 암호화 | 송신자가 책 페이지 x와 단어의 인덱스 y를 전송함 |
        | 복호화 | 수신자는 책 페이지 x의 y번째 단어를 확인하여 메시지를 획득함 |
        | 사전 배경 | 송신자와 수신자가 동일한 책으로 암﹒복호화를 수행해야 하기 때문에 사전에 책을 공유해야 함 |

## 다중 문자 치환 암호(Polyalphabetic Substitution Cipher)
* 복호화를 위한 치환의 대응 관계가 **일대다 대응**임
    - 평문의 한 문자가 암호문에서 여러 종류의 문자로 치환될 수 있음

<br/>

* 다중 문자 치환 암호의 예시
    - 비제네르 암호(Vigenere Cipher) = 비즈네르 암호
        | | 설명 |
        |---|------|
        | 암﹒복호화 원리 | 미리 정해진 키워드(= Key)를 송신자와 수신자가 사전에 공유한 후 이 키워드를 통해 이루어짐 |
        | 암호화 과정 | 1. 비제네르 표에서 키워드의 각 문자에 해당하는 '행'을 고름 <br/> 2. 그 후 키워드의 각 문자를 반복하여 키워드의 각 행에서 평문의 문자에 대응되는 문자들로 평문을 치환함 |
        | 해독 방법 | 키의 길이를 알아내고 이를 도수 분석(frequency analysis, 빈도 분석)을 통해 실제 키를 알아냄 |

## 전치 암호(Transposition Cipher)
* 평문을 구성하는 문자들의 순서를 재배열하여 암호문을 생성하는 암호
    - 평문을 정해진 길이의 블록들로 나누고, 규칙을 적용하여 블록 안의 문자들을 재비치함
      <br/><br/>
      <img width="2030" alt="전치 암호" src="https://github.com/augustf86/Today_I_Learn/assets/122844932/68a8f51a-b335-46bf-83c8-ceee58e6c935">

<br/>

* 전치 암호 예시
    - 스키테일 암호(Scytale Cipher) - 기원전 450년에 고대 그리스인들이 사용
        | | 설명 |
        |---|------|
        | 키(도구) | 나무봉(Scytale)을 공유함 (같은 두께의 나무봉을 사용해야 함) |
        | 과정 | 1. 메세지를 교환할 두 사람이 같은 크기의 나무봉을 제작함 (키 사전 공유) <br/> 2. 송신자는 종이 테이프를 나무봉에 감고 테이프 위에 세로로 메시지를 기입하여 암호문을 만듦 <br/> &nbsp;&nbsp; - 종이테이프를 풀면 순서가 뒤섞여 메시지를 읽을 수 없음 (같은 나무봉이 있어야 복호화 가능) <br/> 3. 종이테이프를 받은 수신자는 같은 크기의 나무봉을 가지고 있으므로 테이프를 다시 나무봉에 감아 이를 해석함 |
